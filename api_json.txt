Un API es una sección dentro del backend que permitirá que 
tanto el Frontend como Backend estén en comunicación.
Tal como la carrocería de un auto se ensambla con el motor.

Existen dos grandes standares/protocolos para construir API:
-soap
-rest

En el protocolo SOAP, el lenguaje que "mueve" la info entre Backend y Frontend es XML 
(parecido al HTML) - permite que se comparta info entre diferentes partes del software.
SOAP es ya tecnología Legacy - ha quedado un poco en el olvido

El protocolo llamado REST es el usado actualmente y es muy superior.
Para hacer un API en  REST, usamos el lenguaje JSON /Java Script Object Notation.
Un Json es un Diccionario de Python :D 
con llaves y valores
Y de hecho lo que para python es un diccionario, para JavaScript es un Objecto

Términos hablados en la clase:

API —> Application Programming Interface
SOAP —> Simple Object Access Protocol
REST —> Representational State Transfer
XML —> Extensible Markup Language
JSON —> JavaScript Object Notation
